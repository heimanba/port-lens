# port-lens：一眼看清端口上在跑什么

本地服务一多，最常卡住的不是写代码，而是：**这个端口到底被谁占了？**  
**port-lens**（命令名 **`ports`**）在终端里用彩色表格列出本机 **TCP 监听**，并尽量把「谁、在哪、什么项目、是否容器」一次说清楚——不用再对着 `lsof` 自己拼信息。

适用于 **macOS**（推荐）与 **Linux**。灵感与定位上与社区工具 [port-whisperer](https://github.com/LarsenCundric/port-whisperer) 相近。

---

## 使用前请确认

- 系统为 **macOS** 或 **Linux**（不支持 Windows）。
- **macOS** 依赖系统自带的 `lsof`。
- 若本机 **Docker** 正在运行且你有权限访问守护进程，工具会把 **发布端口与容器名** 对得更准；没有 Docker 也能正常使用核心功能。

---

## 安装

你需要能在终端里直接执行 **`ports`**。常见做法：把名为 `ports` 的可执行文件放到已在 `PATH` 里的目录（例如 `~/.local/bin/`），然后新开一个终端窗口。

装好后先运行：

```bash
ports --help
```

所有子命令和参数说明都在帮助里。

---

## 怎么用

### 看当前在监听哪些端口

```bash
ports
```

默认会偏向「本地开发常见」的监听，略去一部分桌面/系统噪声。

要看 **本机全部 TCP 监听**：

```bash
ports --all
```

### 盯某一个端口

把端口号换成你关心的数字，例如 **3000**：

```bash
ports 3000
```

会展开该端口上的进程树、工作目录、Git 分支、内存等信息；在合适的情况下也会提示是否结束该监听（按界面说明操作即可）。

### 结束占用端口的进程

结束 **一个或多个端口** 上的监听（请确认没有重要服务在跑）：

```bash
ports kill 3000 5173
```

需要指定 **PID** 时用 `--pid`；若普通结束无效，可用 `-f` 或 `--force` 发送 **SIGKILL**（更强硬，请慎用）。

### 看「像在干活」的进程一览

```bash
ports ps
```

列表里会带上 CPU、内存、框架线索、简短描述等，方便判断是不是昨天没关的本地服务。

不限于常见本地服务类型时：

```bash
ports ps --all
```

### 清理落单的本地服务进程

```bash
ports clean
```

只针对工具内置的 **安全白名单** 里的开发类运行时，用来收拾「孤儿」或僵尸进程；使用前仍建议你看一眼将要结束的是什么。

### 实时看端口变与不变

```bash
ports watch
```

进入 **TUI** 界面，端口新增或释放时会更新显示，适合在起停多个服务时盯着看。

---

## 小结

| 你想做的事 | 命令 |
|------------|------|
| 快速扫一眼常见本地监听 | `ports` |
| 看全部 TCP 监听 | `ports --all` |
| 深挖某个端口 | `ports <端口号>` |
| 关掉指定端口上的监听 | `ports kill <端口…>` |
| 看进程列表（带资源与线索） | `ports ps` / `ports ps --all` |
| 按白名单清理孤立进程 | `ports clean` |
| 实时监控端口变化 | `ports watch` |

更细的参数与边界行为以 **`ports --help`** 为准。项目说明与构建方式见仓库根目录 [README.md](README.md)。
